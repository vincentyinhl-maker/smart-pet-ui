# Heybo Smart Pet 健康管理系统 - 产品需求文档 (PRD)

**版本**: V1.0 (已发布)
**状态**: 归档
**作者**: Antigravity AI Dashboard Team
**最后更新**: 2026-04-06

---

## 1. 产品愿景与定位
将智能硬件（喂食器、饮水机、智能猫砂盆）从孤立的“控制工具”转变为一体化的“宠物健康全景仪表盘”。通过对生理数据的深度挖掘，实现疾病早期预警（急慢性鉴别），保护“忍痛高手”猫咪的健康。

## 2. 核心人群与痛点
- **用户**: 关注宠物精细化健康管理的猫主人。
- **痛点**: 
  - 猫咪生病不外露，发现时往往已是晚期。
  - 多设备数据分散，无法直观对比。
  - 缺乏针对特定品种的科学参考标准。

## 3. 核心功能模块 (V1.0 实现情况)

### 3.1 宠物档案与精准标准
- **品种数据库**: 内置 16+ 种热门猫咪（如布偶、缅因、孟加拉豹猫等）的生命周期生理标准。
- **动态标准**: 系统根据品种及年龄阶段，自动匹配体重、食量、饮水量的建议区间（Standard Range）。
- **个性化档案**: 支持上传宠物头像、自定义名称、记录生日。

### 3.2 多维健康监控图表
- **猫砂盆趋势**: 监测排尿/排便频次及重量变化，关联泌尿系统风险。
- **饮食饮水联动**: 追踪每日摄入量与标准值的偏差。
- **全生命周期成长曲线**: 实时对比实测体重与品种标准区间，监测肥胖或过度消瘦风险。

### 3.3 AI 健康分析预警引擎 (核心逻辑)
系统通过分析 4 大指标（体重、食量、饮水量、如厕频率）识别以下 5 大疾病模式：

1. **慢性肾脏病 (CKD)**: 识别“多饮 + 体重下降”模式。
2. **糖尿病**: 识别“多饮 + 多食 + 体重下降”模式。
3. **甲状腺功能亢进**: 识别“多食 + 体重下降”模式（排除糖尿病）。
4. **FLUTD / 尿路梗阻**: 识别“食量减退 + 频繁蹲厕”模式（区分急迫度）。
5. **慢性胰腺炎**: 识别“食量减退 + 饮水不降反升”模式。

### 3.4 急慢性鉴别诊断
- **慢性建议 (Chronic)**: 实测值偏离标准范围，但与前 7 日私有基线接近。提示调整饲养或预约体检。
- **急性预警 (Acute)**: 实测值不仅偏离标准，且较前 7 日基线出现显著突变（≥22%）。触发紧急警报，建议立即就医。

## 4. UI/UX 设计规范
- **风格**: 暗黑风格“毛玻璃 (Glassmorphism)”特效。
- **动效**: 响应式图表 (Recharts)、微交互反馈。
- **演示功能**: 内置“当日传感器数据面板”，支持手动模拟数据以验证预警逻辑。

## 5. 技术栈
- **前端**: React 18 / Vite 5 / Tailwind CSS
- **图表**: Recharts
- **后端**: Express.js / Node.js
- **部署/归档**: GitHub Release V1.0.0

## 7. V2.0 平台化升级核心需求 (Pet-Centric IoT Platform)
*以下为 V2.0 阶段架构演进与平台开放设计需求：*

### 7.1 “以宠物为主线”的长期数据模型设计
- **设备与宠物解耦**：底层数据彻底从“设备产生数据”转移为“产生属于特定宠物的数据”。设备（喂食器、猫砂盆）可能损坏、更换，但宠物的健康档案（寿命达 10~20 年）永不间断。
- **长效历史数据链路**：单只宠物支持记录上万天的数据。UI 层展示 180 天、360 天数据时，需要高效的降采样/汇总表（Rollups）支持，以平均值展现长期趋势。

### 7.2 断点数据自动修补机制 (Data Imputation)
- **场景**：如果由于设备断电、断网或维修导致某天或某几天数据缺失，系统需具备“空值插补（Imputation）”机制。
- **算法设定**：通过临近时间点（如前后 3-7 天）的滑动平均值（Moving Average）来推算替代缺失值，保证 360 天健康走势图的平滑并且符合生理稳定态。

### 7.3 海量设备并发与云端架构适配
- **高并发支持**：V2.0 系统设计需支持单台节点 **上万至十万级** 设备同时在线。
- **数据库拓展性**：架构虽在初期开发测试时基于本地轻量级数据库（SQLite），但数据结构模型需支持无缝迁移至企业级关系型/时序数据库（如 PostgreSQL / TimescaleDB），确保存储体量升级能力。
- **从轮询到事件驱动**：与涂鸦 IoT 平台的交互由目前的定时拉取（Poller）升级为支持**涂鸦消息订阅 (Pulsar / Webhook)**，实现高吞吐、低延迟状态同步。

### 7.4 平台级指标扩展性 (Open Ecosystem)
- 当前接入的是 Heybo 产品体系。系统需预留一套标准化的**设备/数据点网关层 (Device Gateway)**。
- 允许未来随时增加新类型指标：**运动量**（智能项圈产生）、**体温**、**心率血氧**等。新增指标时无需大规模重构底层表，依靠通用的时序指标表 (Time-Series Metric Tables) 进行字段扩充或 EAV (Entity-Attribute-Value) 模式扩展。
