# 智能宠物用品移动端APP - 产品需求文档 (PRD)

**状态**: 初稿
**作者**: 资深产品经理 (Product Manager)
**产品名称**: SmartPet (暂定)

## 1. 目标用户分析
**核心人群**: 养宠物（主要为猫、狗）的都市白领 (22-35岁)。
**用户画像**:
- 工作繁忙，白天长时间离家，无法按时喂养宠物。
- 有较高消费能力，愿意为宠物的健康和陪伴付费。
- 对产品外观和App交互体验要求较高，偏好极简现代设计。
- 痛点：担心宠物饿肚子、想知道宠物在家是否捣乱/安全、设备配网麻烦。

## 2. 核心功能点
1. **设备鉴权与连接**：通过账号系统登录，快速绑定及解绑智能喂食器/摄像头。
2. **远程实时控制**：随时随地一键手动出粮。
3. **宠物状态监控**：实时查看设备摄像头画面（支持流媒体渲染设计）。
4. **喂食与状态记录**：查看历史喂食记录（时间、出粮克数等）。
5. **异常消息提醒**：设备缺粮卡粮、宠物异常活动时的即时推送提醒（二期或设置页基础展示）。

## 3. 用户旅程图 (User Journey)
1. **注册/登录**: 用户首次下载App -> 手机号或邮箱注册 -> 登录成功，进入无设备的空白主页。
2. **设备主页日常查看**: 午休期间 -> 掏出手机打开App -> 在设备主页看到“正在连接设备” -> 连接成功，看到剩余余粮 60%、摄像头小窗在线、上次喂食时间。
3. **远程喂食**: 感觉宠物饿了 -> 点击“手动喂食”或进入远程控制页 -> 设定10g -> 点击“出粮” -> 界面弹出“出粮成功”反馈动画 -> 历史记录+1。
4. **设置与修改**: 晚上下班后 -> 打开设置页 -> 修改个人资料或查看设备通用信息。

## 4. 功能优先级列表 (MoSCoW法则)

### M (Must Have) 必须有
- 用户注册与登录
- 设备主页及基础设备状态展示 (在线/离线/余粮)
- 远程单次手动喂食操作
- 历史喂食记录列表

### S (Should Have) 应该有
- 视频监控推流界面嵌入
- 设备添加与设置管理

### C (Could Have) 可以有
- 定时喂食计划设定（本次范围可简化并放在后续迭代）
- 白天模式/夜间模式切换

### W (Won't Have) 本次暂不包含
- 宠物社区功能、电商购买功能
- 复杂的人脸或宠物行为AI分析功能
